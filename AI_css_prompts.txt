Thought for a couple of seconds

That’s happening because your CSS sets the banner image to a max-height with object-fit: cover, which crops the image to fill the space. In your style.css you have:

header img{
  width:100%;
  max-height:240px;
  object-fit:cover;
  border-radius:12px;
}

cover = “fill the box, even if I have to cut off edges.”

Fix (show the whole banner, no cropping)

Replace your header img{...} rule with this:

header img{
  width: 100%;
  height: auto;          /* lets it keep full proportions */
  max-height: none;      /* no forced cropping height */
  object-fit: contain;   /* shows the whole image */
  border-radius: 12px;
  display: block;
}
If the banner becomes “too tall”

Use a gentler limit that still doesn’t crop, like this:

header img{
  width: 100%;
  height: auto;
  max-height: 320px;     /* adjust if needed */
  object-fit: contain;   /* no cropping */
  border-radius: 12px;
  display: block;
}
Why your banner looked cut off

Your banner is wide, and the CSS forced it into a short 240px-tall box.

object-fit: cover chopped off the top/bottom to make it fill the box.

Let’s fix it in a “can’t be overridden” way.

1) Replace your header img { ... } rule with this (use exactly this)

In your current style.css you already have a header img block.
Replace that entire block with:

header img{

  height: auto !important;
  max-height: none !important;
  object-fit: contain !important;   /* show whole image, no cropping */
  border-radius: 12px;

}
2) Add this to your header { ... } rule to stop any container clipping

Right now your header has no overflow, but adding this prevents any accidental clipping from rounding/shadows/etc.

header{
  border-radius: var(--radius);
  background: rgba(255,255,255,.05);
  border: 1px solid rgba(255,255,255,.1);
  overflow: visible !important;   /* do not clip banner */
}
